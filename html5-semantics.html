HTML Semantics Site

Topics: Front End Development: HTML5 Semantics: Sept. 29th 2013

HTML5 Semantics are code elements that clearly describe the structure of your site to the browser.

Things I Learned

  • Semantic elements outline the different parts of a web page
  • A header is a semantic element that creates a heading for the site. You can have several header elements in one document
  • An Aside is a semantic element that helps define content as a sidebar statement. Allows for more explanation of text
  • The nav semantic element allows for a reference of a web link on your page
  • Other semantic elements include article: figure: footer: section: and time to name a few

Summary

HTML5 Semantic Elements act as the building blocks for the structure of your web page. These codes create segments were text is inputed to define your page: establish links: state date and time: create side notes as asides: and specifies self contained elements such as illustrations, diagrams, photos etc.
